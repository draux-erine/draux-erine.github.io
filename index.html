Project 1 : Database development and visualisation

Project Context

The purpose of this project was to clean, and analyse data of an open database: OpenFoodFact by making reports to give precise insights on Nutri-Score. Once done, another aim was to use Machine Learning processes on Knime to determine the Nutri-Score of certain products. The main difficulty resided in managing a vast dataset characterized by a significant number of incomplete entries and missing values, which presented a major challenge to delivering relevant and proper analyses.

The results

Our analysis revealed several key insights about the factors influencing Nutri-Score ratings :

This infographic shows that products with many additives usually have poor Nutri-Scores (mostly D or E), while products without additives tend to score better. It also highlights that non-nutritive sweeteners are linked to higher Nutri-Scores because they add almost no calories.

The impact of packaging and labels on nutritional quality was also studied. Plastic and sachet packaging contained proportionally more products with poor Nutri-Scores (D and E). Likewise, most “positive” labels (such as Sustainable Trade or Vegetarian/Vegan) did not correspond to better nutritional quality and were often linked to low Nutri-Scores. Howerver labels related to composition or production such as Organic or Reduced Fat/Sugar showed a more balanced nutritional profile and higher Nutri-Score.

My Role

My role was polyvalent throughout the project. Indeed, I actively participated in the design of the Conceptual Data Model (CDM), ensuring the data structure was optimized for our analysis. I then took the initiative to deepen my skills by handling the data cleaning and integration into the database, where I evaluated different approaches using both Python scripts and Knime workflows to determine which one was more practical for me. Furthermore, I was made responsible for creating the analysis reports on the impact of labels and packaging on the Nutri-Score. Facing the challenge of a large volume of incomplete entries and missing values, I demonstrated rigor and a problem-solving mindset by implementing filtering strategies to overcome this problem and successfully deliver reliable insights. Finally, I collaborated in the modeling phase by using distincts Machine Learning processes on Knime for Nutri-Score prediction. This project not only developed my technical skills but also enhanced my transversal skills, especially my autonomy when dealing with raw data, my problem-solving ability, and my team spirit.

Tools And Technical Approach

I used both Python and Knime for scripting and data cleaning, and for advanced data integration, processing, and the application of Machine Learning models, I used the visual workflow platform Knime. The data itself was sourced from the OpenFoodFacts API, structured using the Conceptual Data Model (CDM) methodology, and managed within a PostgreSQL database. For predictive modeling, I evaluated differents Machine Learning algorithms, including Gradient Boosted Trees and Multi-Layer Perceptron, to predict the Nutri-Score. Finally, Power BI was used for data visualization and the creation of detailed analytical reports.

Project 2 : Conception and implementation of an application

Project Context

The core challenge of this project was to conceive and develop a real estate management application based on client expectations and specifications. The initial difficulty laid in the analysis of the subject and needs, which required many exchanges with the client to fully grasp the complex business rules of the real estate sector. This led to the fundamental necessity of defining a clear Conceptual Data Model (CDM), which served as the essential blueprint for the application's implementation.

What I Delivered

The final outcome was a fully functional application developed in Java which passed all of the tests scenarios given. Beyond the successful delivery of the product itself, the project resulted in a significant reinforcement of my knowledge of Object-Oriented Programming (OOP) and my ability to set up robust software architectures. My Role

I was an active member of a five-person development team, and my involvement covered the entire project, from the initial subject analysis to the final application delivery. In the planning phase, I worked with a teammate to define the User Stories and establish the functional specifications, making sure all user scenarios matched our application’s requirements. From a programming perspective, I was responsible for developing several key features focused primarily on the View and Controller (MVC) layers. This included creating the functionalities to create, display, and list buildings and various company types (such as construction, repair, and insurance companies). I also implemented important financial logic, including the ability to add expense provisions when creating new properties and generating the complete expense summary report. Furthermore, I designed and built the dedicated window for managing (adding or deleting) insurance policies. Finally, I contributed to continuous code improvement efforts to maintain quality, and I was essential in producing the final user guide.

Database replication in a redundant system

INEO ES

April — June 2024

The company’s redundancy system experienced data inconsistencies during server failover. The main goal of this internship was therefore to enhance the system resilience. My mission involved identifying and implementing a database replication solution.

In the end, I successfully identified two viable data replication solutions suitable for the company's two-server infrastructure. The initial research and testing phase resulted in the elimination of multiple incompatible tools (including specific database and synchronization mechanisms), which defined the final scope of deployment. I then focused on the implementation of one solution, creating the necessary Java integration code and performing functional tests to confirm its ability to replicate data and maintain consistency during failover events. I delivered a comprehensive technical analysis, code samples, and architectural diagrams. These materials provide the internal team with the practical groundwork required to deploy the chosen replication system, ensuring the coherence of critical alarm data and supporting service availability.

My role during this three-month internship was centered on improving data consistency in the redundant server environment. I was responsible for conducting the comparative study and hands-on testing of various database replication solutions. After selecting the most suitable mechanisms, my primary contribution involved developing the Java code for a to integrate the chosen system with the existing application architecture. I ensured detailed documentation of the findings and the implementation steps, delivering the technical foundation for the project’s next phase.

This project required applying and expanding expertise across several key technologies. The implementation was conducted entirely within a Linux environment, utilizing Ubuntu Virtual Machines for system setup and testing. I gained hands-on experience with several database technologies including MongoDB, SQLite, and Redis, and focused heavily on evaluating various data replication mechanisms (Master-Replica models). The integration work involved Java development (utilizing class diagrams and adhering to best practices) to interface with the core application. For advanced troubleshooting, I used Wireshark for deep TCP exchange analysis, and relied on application logs and tools like JConsole to monitor system behavior and validate data consistency.

My commitments

Digital Fresk & Climate Fresk

Digital Fresk

The Digital Fresk (or La Fresque du Numérique) is an engaging and collaborative workshop created to raise awareness about the environmental and societal challenges of digital technology. The goal is to work together to organize cards detailing the causes and consequences of digital impact (from device manufacturing to data center usage). It allows to gain a comprehensive understanding of the issue and collectively identify tangible actions and solutions in order to build a more sustainable and responsible digital world.
